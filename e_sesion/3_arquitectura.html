Arquitectura

Cuando un PWA necesita que el servidor procese información, debe invocar servicios alojados en dicho servidor.

Cada servicio es una función y para accederlo, el servidor le asigna un nombre de recurso, el cual es una ruta dentro de la URL; es como si fuera el nombre del archivo. Por ejemplo, en la URL http://localhost:3000/validaUsuario, el nombre del recurso es /validaUsuario.

En este caso, a todos los archivos de HTML y JavaScript les llamaremos www.

Para validar el acceso usaremos un servicio llamado validaUsuario que tiene asociado el nombre de recurso valida.

Para poder validar el acceso, validaUsuario usa una entidad de base de datos llamada Usuario, donde se guarda la contraseña de cada usuario.
